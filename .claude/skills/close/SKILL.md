# close

Зафиксировать завершённую работу: задача → `done` + строка в лог сессии проекта. Это ритуал, который держит отслеживаемость честной. Самодостаточно: можно дать любому агенту.

## Когда вызывать

- Завершил задачу из беклога → закрой её.
- **Сделал что-то значимое, для чего задачи ещё не было** (фича, фикс, деплой, новый файл/страница/скилл, редизайн, миграция — в том числе в инфра/мета-проектах: сам vault, дашборд, скиллы) → заведи задачу постфактум (см. `backlog`) сразу со `status: done` и закрой здесь же. **Не отчитывайся пользователю о готовой работе, пока она не отражена задачей.**

## Шаги

1. **Найти/создать задачу.** Есть в беклоге — взять её. Нет — создать по контракту навыка `backlog`, но сразу `status: done` (а не `todo`); RICE всё равно проставить (для истории и метрик).
2. **Перевести в done.** В frontmatter: `status: done`, `updated: <сегодня>`. Отметить выполненные пункты DoD (`- [x]`).
3. **Записать в лог сессии проекта.** В `projects/<slug>/<slug>.md` — в начало раздела с логом сессий добавить (или дополнить сегодняшнюю секцию) строкой: что сделано + ссылка `[[tasks/<имя>]]`. Формат секции: `## Что сделано в сессии <YYYY-MM-DD>`.
4. **Обновить роадмап**, если задача закрывала пункт фазы (`- [x]`).
5. **Commit + push** (если git).
6. Коротко отчитаться: какая задача закрыта (`id`), что записано в лог.

## Пример

Только что добавил на дашборд поиск по задачам, задачи на это не было.
→ создать `projects/dashboard/tasks/task-search.md` со `status: done`, DoD «на /search есть живой поиск по id/названию/тексту», RICE проставлен;
→ в карточке `dashboard` в лог сессии за сегодня дописать: «Поиск по задачам на /search ([[tasks/task-search]])»;
→ commit, push;
→ ответить: «Закрыла LLMUSH-154 (поиск), записала в лог».

## Правила

- Закрытие без записи в лог сессии не считается полным — карточка проекта должна отражать, что сделано.
- Не удалять задачу при отмене — ставить `status: cancelled` (история решений тоже память).
- Язык — русский.

## Самопроверка

- [ ] У завершённой работы есть задача (нашёл или создал постфактум)
- [ ] `status: done`, `updated` = сегодня, DoD отмечен
- [ ] В карточке проекта есть строка в логе сессии со ссылкой на задачу
- [ ] (если git) запушено
