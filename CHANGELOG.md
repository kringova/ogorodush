# Changelog

Заметные изменения методологии и навыков. Новое — сверху.

## v1.4.0 — 2026-07-08 — Навык update и smoke-CI шаблона

### Добавлено
- **Навык `update` («обнови Огород»)** — путь от баннера обновления к применённому обновлению: доставка канона с тега релиза (git clone / tar-фоллбэк), диф-превью с подсветкой локальных кастомизаций, применение только канонных путей (`_rules.md`, `skills/`, `docs/`, `_templates/`, `scripts/`, `.githooks/`), пользовательский контент не трогается, `ogorod_version` записывается после. Проверен живым обновлением клона v1.1.0 → v1.3.1.
- **Smoke-CI шаблона** (`.github/workflows/smoke.yml`): на каждый push — мини-дашборд отвечает контентом примера, dashboard-app собирается и рендерит, pre-commit-гейт блокирует `done` без апрува. Бейдж в README.

## v1.3.1 — 2026-07-08 — Хвост аудита: приватность коммитов, ребренд дашборда, дока

### Исправлено
- **Приватность истории**: `scripts/agent-commit.sh` псевдонимизирует человека GitHub-noreply-адресом владельца origin (личный email и hostname машины больше не попадают в коммиты; явный override — env `OGOROD_HUMAN`).
- **Ребренд дожат в full-дашборде**: тикеты `OGOROD-####` вместо `ARTEL-####` (единая константа `TICKET_PREFIX`), cookie `ogorod_session`, HMAC-соль `ogorod-dashboard-v1`. Дашборд разлогинит один раз.
- `VAULT_PATH`: понятная ошибка вместо тихого фоллбэка, если переменная не задана и запуск не из корня vault.
- Дока = реальность: описаны навыки `setup` и `charter` (docs/skills, architecture, README, чеклист installation); architecture.md актуализирован про грядки и dashboard-app; `jtbd` в таблице frontmatter проекта; `_rules.md` — коммит явными путями вместо `git add -A` + документирован гейт `check-staged-scope`; пример проекта получил `jtbd`; `.DS_Store` вычищен из индекса.

## v1.3.0 — 2026-07-08 — Дайджесты и ручной трекер в дашборде

### Добавлено
- **Навык `recap` → «Дайджест»**: умный нарратив за период (неделя / месяц / полугодие) — эффект вместо списка задач, разделы «Свернули» и «Куда дальше». Сохраняется файлом в `<грядка>/digests/`; `sync` сам замечает закрывшийся без дайджеста период и генерирует недостающий.
- **Грядки в дашборде** (dashboard-app): переключатель грядок, страница `/beds`, bed-aware чтение vault и API.
- **Ручной трекер**: регулярные задачи (модель повторов + отображение) и личная аналитика ручной грядки — вместо тиринга.

### Изменено
- `scripts/agent-commit.sh`: атрибуция коммита — автор = модель сессии, человек из git config → соавтор (ровно две личности).
- Ребренд функциональных идентификаторов (env, конфиг) под Огород.
- `docs/reference.md`: документированы опциональные `cost_*`-поля метрики тиринга (заполняет адаптер).

## v1.2.0 — 2026-06-26 — Грядки: пространства внутри Огорода

### Добавлено
- **Концепт «грядка»** (`docs/methodology.md` → «Грядка»): пространство внутри единого Огорода (один репо, один интерфейс). Два типа по контракту авторства: **нейрогрядка** (`type: agent`) — контент пишет агент, наследует агентский слой (роли, тиринг, гейт `review→done`); **ручная грядка** (`type: user`) — ведёт человек через UI, агент контент не пишет.
- Схема `bed.json` (`id`, `name`, `type`; `visibility` зарезервировано).
- Файловая архитектура: `beds/<грядка>/projects/<проект>/tasks/<задача>.md`; принципы легибельности без UI.
- Поблочное редактирование md: HTML-маркеры-якоря `<!-- block:id -->…<!-- /block:id -->` для безопасной частичной записи.
- **Личный RICE** для ручной грядки: `приоритет = (Важность × Срочность / Сложность) × (1 + (Интерес−1)/14)`; дефолт 4 по всем осям.

## v1.1.0 — 2026-06-18 — Уведомление об обновлениях шаблона

### Добавлено
- **Огород сам сообщает об обновлении.** В начале работы `sync` сравнивает версию vault'а (`ogorod_version` в `ogorod.config.json`) с последним GitHub-релизом Огорода и при новой версии выводит баннер «🌱 Обновление Огорода» с описанием из релизноутс. Проверка best-effort и read-only: офлайн или ошибка — молча пропускается, sync не ломается. Применить обновление — отдельной командой «обнови Огород».
- `setup` записывает `ogorod_version` в `ogorod.config.json` при установке.

### Изменено
- Ребренд: **Артель → Огород** (`artelush` → `ogorodush`). Позиционирование — «рабочая среда для проектов с ИИ-агентом».

## 2026-06 — `sync` подсвечивает инбокс

### Добавлено
- **`sync` считает и подсвечивает инбокс**: навык теперь читает `_inbox/`, считает необработанные записи (кроме `.gitkeep`) и при N > 0 выводит блок «📥 Инбокс» с предложением запустить `/inbox`. Закрывает боль «инбокс выпадает из внимания между сессиями», не нарушая read-only-инвариант `sync` — **разбор не втягивается**, остаётся отдельным навыком `inbox` и явным решением пользователя.

## 2026-06 — Навык `onboard` (brownfield-вход)

### Добавлено
- **Навык `onboard`** (`skills/onboard/SKILL.md`): поднимает первичный канон документации существующего проекта при переходе на Огород. Приоритет источников по APC-Collaboration: артефакты (ТЗ, заметки, тикеты, старые доки) → код (стек/команды/структура) → человек-SME **последним**, только для пробелов и противоречий. Выведенное помечается «черновик, проверь», пробелы — `TODO(SME)`; APC задаёт честную границу — догадка не выдаётся за факт. Прямой ответ боли «нужна дока, не знаем с чего начать».

## 2026-06 — Вендор-агностичность конфига агента

### Добавлено
- **Конвенция переносимого конфига** (`docs/doc-canon.md` → «Вендор-агностичность конфига»): `AGENTS.md` — канонический контракт в корне каждого репо (читается Codex/Cursor нативно); `CLAUDE.md` = `@AGENTS.md`-шим; вендорские папки (`.claude/` и пр.) — read-only зеркала в `.gitignore`, конфиг правится только в каноне. Полноценный компилятор `.agents/` → форматы вендоров отложен до входа второго вендора в ежедневную работу. Обязательна для новых проектов; существующие — лениво.

## 2026-06 — Гейт закрытия (review-потолок агента)

### Добавлено
- **Гейт закрытия задач** (`scripts/check-task-close.py`, `.githooks/pre-commit`): pre-commit хук механически блокирует агентский коммит с переходом задачи в `done`. Потолок агента — `review`; `done` ставит только апрув-поверхность адоптера через `OGOROD_APPROVE=1`. Установка: `git config core.hooksPath .githooks` (см. `docs/installation.md`).
- Навык `close` переписан под review-потолок: агент переводит в `review` и предъявляет отчёт — `done` ставит человек.

## 2026-06 — Огород: команда ролей и документация

### Добавлено
- **Роли** (`docs/roles.md`): аналитик, разработчик, ревьюер, техписатель — включаются по компонентам RICE; поле `roles` в задаче.
- **APC-канон документации** (`docs/doc-canon.md`, `docs/styleguide.md`): обязательный набор внешней и внутренней доки по зрелости проекта; модель Audience · Purpose · Collaboration; правила против ИИ-воды.
- **Жизненный цикл и гейты** (`docs/workflow.md`): пороги «RICE → роль», гейт «готово», релиз-гейт для всех проектов.
- **Тиринг моделей** (`docs/methodology.md`): грейды `junior/middle/senior` по природе задачи, эскалация; вендор-устойчивость.
- **Правило runbook перед деплоем**: знать процесс деплоя из доки, не реверс-инжинирить; дока обязана соответствовать реальности.
- Навык `recap` — итоги за период.

### Изменено
- **Переименование `taskvault` → Огород (`ogorodush`)**: фреймворк вырос из хранилища задач в команду ролей + документацию + тиринг. Беклог — основа-подсистема.
- `backlog` проставляет `roles` и `model_tier`; `close` ведёт `CHANGELOG`.
- Правила вендор-нейтральны: `AGENTS.md` канон, `CLAUDE.md` — шим.
