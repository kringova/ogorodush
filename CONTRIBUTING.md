# Как контрибьютить в Огород

## Куда что предлагать

- **Методология, доку, `_rules.md`** → папка `docs/` и файл `_rules.md`. Изменение должно соответствовать стайлгайду (`docs/styleguide.md`): лаконично, без воды, активный залог.
- **Навыки** → папка `skills/<навык>/SKILL.md` — самодостаточные инструкции, каждый навык в своей папке с описанием (заголовок, шаги, примеры).
- **Дашборды** → `dashboard/` для mini-версии (HTML без сборки) или `dashboard-app/` для full (Next.js).

## Канон и копии

Этот репо — **канон методологии Огорода**. Адаптеры (llmush, другие проекты) копируют отсюда навыки и инструкции, адаптируя пути в своих контекстах. PR направляйте в канон, а не в копию.

## Правила PR

1. Прочитайте `docs/styleguide.md` — обязательно следуйте канону (против ИИ-воды, лаконичность, APC-шапка для документов).
2. **Дока = реальность.** Меняете поведение — обновите дока в том же PR, не отдельно.
3. Заметное изменение → одна строка в `CHANGELOG.md` с кратким описанием.
4. Breaking-изменение (схема frontmatter, структура папок, формат конфига) → секция «Миграция» в `CHANGELOG.md` и release notes с конкретными шагами для пользователей; мажорная версия по semver.
5. CI должен быть зелёным (smoke-тесты).

## Релизы

Мейнтейнер публикует **GitHub Release** с notes (скопированы из `CHANGELOG.md`). Пользователи получают уведомление через `sync` — версия в CHANGELOG без релиза для них не существует.
