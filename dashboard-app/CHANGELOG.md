# Changelog

Заметные изменения дашборда. Новое — сверху.

## 2026-06

### Добавлено
- **Грядки (beds) в UI** (#383) — дашборд работает с несколькими грядками: переключатель в шапке, страница `/beds` (список, создание, переименование), активная грядка в `?bed` (разные грядки в разных вкладках). Чтение vault стало bed-aware (проекты/задачи/инбокс из папок активной грядки); корневая грядка — поведение по умолчанию. Концепт грядок описан в `docs/methodology.md` — теперь виден и в UI.
- **Сводка за сегодня** на `/analytics` (#261) — наверху страницы токены по грейдам/моделям и измеренные задачи только за сегодня (по дате закрытия из git), с переключателем «сегодня / за всё время».
- **Аналитика тиринга** `/analytics` (#204, #209, #210, #214) — стоимость 1 SP в токенах по грейдам (хитмапа), velocity, SP по проектам, динамика % junior.
- **Токены по грейдам и моделям** (#215) — виджет группирует токены по грейду фактической модели (junior/middle/senior) с долей-полоской, внутри — конкретные модели. Грейд берётся из того, кто реально делал работу, а не из плановой разметки задачи; матрица стоимости SP считает так же.
- **Короткий URL задачи** `/t/LLMUSH-0190` (#194) — канон, старые длинные URL → 308.
- **URL задач по ключу** `LLMUSH-####` вместо имени файла (#190); роут резолвит и по ключу, и по старому slug.
- Дневная активность в шапках канбана: создано у Todo, закрыто у Готово (#180).
- Виды: карточки проектов (с прогрессом и секциями), канбан, RICE-приоритеты, аналитика, инбокс, плюс роадмап (на странице проекта) и сгорание (виджет в аналитике).

### Изменено
- **Сгорание — в Story Points и событийно** (#251): было методологически криво — считалось число задач (а не объём работы в SP), и все сейчас-открытые задачи учитывались как открытые с начала истории, отчего кривая фиктивно висела у максимума и резко падала в конце. Теперь остаток = сумма SP открытой работы, события `+SP` при создании задачи и `−SP` при закрытии — кривая отражает реальный рост объёма и сгорание. Легаси-задачи без SP в график не входят.
- **Графики аналитики в едином стиле** (#261): «Динамика грейдов» приведена к виду «Сгорания» — та же высота, мягкая заливка под линиями, точки, сетка и подписи осей. У столбцов «Покрытия тиринга» при наведении — точный процент дня.
- **Навигация чище — 9 вкладок → 5** (#238, #211, #216): шапка теперь Проекты · Канбан · RICE · Аналитика, «Справка» и «Инбокс» вынесены вправо. Поиск переехал строкой + фильтрами (проект, грейд) на канбан (вкладки «Поиск» нет, `/search` → `/kanban`). Роадмап стал секцией на странице проекта (отдельной вкладки нет). «Сгорание» теперь виджет внутри «Аналитики».
- **Грейд везде одним бейджем** (#213): аватар-кружок с буквой (J/M/S, цвет по грейду) вместо разнобоя пилюль и лозенгов — на карточках, в фильтре канбана, в аналитике. У **закрытых задач** грейд показывается по фактической модели (кто реально делал), на странице задачи видно сами модели. В Справке — раздел «Грейды» с профилями.
- **Инбокс — иконка с каунтером** (#240): «Инбокс» в шапке стал иконкой письма справа с числом задач, ждущих апрува (`review`); 0 — без значка. Очередь «На подтверждение» видна из любого экрана.
- **Виджет «Сгорание» опрятнее** (#238): цифры без коробок-в-коробке, точки графика равномерно по ширине (не сбиваются по краям), компактнее по высоте.
- **Динамика грейдов — накопительная** (#261): «Доля грейдов по задачам» заменена на «Динамику грейдов (накопительно)» — доля senior/middle/junior считается нарастающим итогом по 3-часовым бакетам реального времени, видна траектория и паузы в работе; последняя точка совпадает с итогом «за всё время».
- **Покрытие тиринга — столбцы по дням** (#261): вместо горизонтальных баров — вертикальные столбцы (высота = все закрытые за день задачи, цветом — доля с залитыми токенами), не уезжают вниз по мере накопления дней.
- **Доля грейдов — из задач, не из снимков** (#236): виджет «Доля грейдов по задачам» показывает долю senior/middle/junior в токенах **по каждой задаче** во времени (со скользящим средним) — видно, как тиринг меняется от задачи к задаче. Ряд реконструируется напрямую из задач, дашборд больше не зависит от суточных снимков/крона.
- **Аватарки грейдов** (#213): 🐣 junior · 🦆 middle · 🦅 senior — круглый аватар в углу карточки, как исполнитель в Jira.
- **Роадмап на странице проекта** (#216): поднят наверх и свёрнут под кат (раскрывается по клику).
- **Мобильная вёрстка** (#174): дашборд адаптирован под телефон. **Нижняя навигация** — бар Проекты·Канбан·Инбокс·RICE·Аналитика с центральной кнопкой-инбоксом и счётчиком ждущих апрува (как в банковских приложениях). Канбан на мобиле — переключатель статусов (одна колонка за раз). Широкие таблицы и Гант скроллятся внутри блоков, страница не выезжает вбок. Десктоп без изменений.
- **«Динамика» — поток за день** (#222): график показывает % junior и токенов/SP **за день** (дельта между снимками), а не накопленный итог — видно, стало ли дешевле именно сегодня. Снимки копит ежедневная облачная роутина (22:00 MSK).
- **Аналитика чище** (#219): виджет токенов переоформлен (крупный % грейда, толстый бар-доля, модели под-строками, компактный cache). Убраны дублирующие виджеты «Velocity» и «SP по проектам».

### Исправлено

- **Починен CI/CD** (#191): workflow лежал в `.git/info/exclude`, автодеплой не работал; возвращён в репо. Добавлен runbook (`deploy/README.md`), порт в README исправлен.
