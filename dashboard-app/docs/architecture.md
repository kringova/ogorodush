# Архитектура ogorod-dashboard

## Принцип

Дашборд **не имеет своей БД**. Источник правды — markdown-файлы vault.
Next.js на лету читает их (`gray-matter` фронтматтер + `remark` тело) и рендерит
виды. Запись только одна — инбокс.

## Поток данных

```
vault (md: projects/*, tasks/*) 
   → src/lib/vault.ts  (парсинг фронтматтера → типы Task/Project, расчёт RICE, key=OGOROD-####)
   → роуты src/app/*   (projects, t/[key], kanban, rice, analytics, inbox, roadmap, burndown, help)
   → src/lib/markdown.ts (рендер тел задач и [[wiki-ссылок]])
```

Путь к vault — env `VAULT_PATH`. Даты «создана/последнее действие» берутся из git
(`src/lib/git.ts`), если нет во фронтматтере.

## Модули `src/lib/`

| Модуль | Ответственность |
|---|---|
| `vault.ts` | чтение vault, типы `Task`/`Project`, RICE, `getTask`/`getProject`; `key` = `OGOROD-####` |
| `markdown.ts` | рендер md-тел, преобразование `[[tasks/...]]` в ссылки |
| `git.ts` | первая/последняя дата коммита файла |
| `ui.ts` | форматтеры: `fmtTicket`, `fmtRice`, статусы |
| `config.ts` | capacity (ч/нед), перевод оценок в календарные сроки |

## Навигация (шапка `components/Nav.tsx`)

Основные вкладки-виды: **Проекты · Канбан · RICE · Аналитика · Инбокс**. Справа
отдельно — **Справка** (`/help`). Активная вкладка подсвечивается (`usePathname`).

Часть видов сознательно НЕ выносится в шапку (минимум шума):
- **Поиск** живёт строкой + фильтрами (проект, грейд) на канбане; роут `/search` → 308 на `/kanban`.
- **Роадмап** — секция на странице проекта (`projects/[slug]`); глобальный Гант `/roadmap` жив, но без вкладки.
- **Сгорание** — виджет на `/analytics`; страница `/burndown` тоже жива (общий компонент `BurndownChart`).

## Роуты `src/app/`

- `projects` — карточки проектов с прогрессом; `projects/[slug]` — проект + задачи + секция «Роадмап».
- `t/[key]` — короткий канонический URL задачи (`/t/LLMUSH-0190`); старые длинные URL → 308.
- `kanban` (поиск + фильтры проект/грейд) · `rice` (приоритеты) · `analytics` (тиринг + виджет сгорания) · `roadmap` (глобальный Гант) · `burndown` (сгорание) · `help` (Справка).
- `search` → редирект на `/kanban` (роут оставлен ради старых ссылок).
- `inbox` + `api/inbox` — захват мысли: форма → файл в `_inbox/` vault → commit + push.
