# Локальный дашборд

Веб-витрина vault без зависимостей и без сборки. Один Node-скрипт читает markdown-файлы и отдаёт страницу: проекты, канбан (todo / в работе / блок / готово), беклог по RICE, инбокс.

## Запуск

Из корня vault:

```bash
npm run dashboard          # откроет http://localhost:4321 в браузере
# или напрямую:
node dashboard/serve.js              # без авто-открытия
node dashboard/serve.js --port 8080  # другой порт
node dashboard/serve.js --open       # открыть в браузере
```

Нужен только Node.js 18+ (никаких `npm install`).

## Как это работает

- Данные читаются из файлов **при каждом запросе** — правки агента видны по обновлению страницы (страница сама перезагружается каждые 5 секунд).
- Парсятся: карточки проектов (`projects/*/<slug>.md` с тегом `project`), задачи (`tasks/*.md` с тегом `task`), записи инбокса (`_inbox/*.md`).
- RICE считается на лету: `(reach × impact × confidence%) / effort`.
- Ничего не пишет — только читает. Закрыли терминал → дашборд остановился.

## Запустить на другом vault

```bash
node dashboard/serve.js --root /путь/к/другому/vault
```

## Чем отличается от Obsidian-сводки

`_dashboard.md` + Dataview работают внутри Obsidian. Этот дашборд — отдельная веб-страница: удобно держать открытой во вкладке, не требует Obsidian, видна с телефона в той же сети (`http://<ip-компа>:4321`).
